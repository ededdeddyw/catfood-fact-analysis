## 1. 公式URL解決（②）— 79社中29社を出典付きで確定

協議会の会員ページは会社名・住所のみで**公式サイトへのリンクが無い**（テキスト掲載）。
そこで **Bingスクレイプ → 候補を実取得して会社名一致を検証** する方式にした。
検証ゲートがあるので、検索が誤爆しても**捏造URLはCSVに入らない**（出典必須の担保）。

- **確定 29社**：主要大手を網羅（アイシア・いなば・日本ペットフード・ヒルズ・ロイヤルカナン・
  ユニチャーム・マース・ネスレピュリナ・ペットライン・ジェックス・ライオンペット・ビルバック 等）。
- **要確認 50社**：中小・輸入の長尾。Bingは `kkmars.co.jp` `life-base.jp` 等の広告/汎用結果を
  繰り返し返し、会社名一致だけでは **役場・旅行・証券サイト等の偽陽性**が出た（INO→いの町役場、
  イースター→るるぶ等は降格済み）。**中小社の公式ドメインはトークンゼロでは解決が難しい**という所見。
  → 残り50社は人手 or 別ソース（協議会への問い合わせ・パッケージ裏・EC実需リスト）で確定する。

`maker_sites.csv` の `method` 列に出自（curated / verified_proposal / manual_known / bing_qN /
rejected_falsepos / unresolved）を残してある。

## 2. 開示率監査（③）— 確定29社を巡回した実測

各社トップから同一ドメインを最大20ページBFSし、保証分析値テキストを抽出。

### 2-1. いちばん重い事実：**大手は静的HTMLで全滅**

- **29社中 23社が成分ページ0件**。アイシア・いなば・日本ペットフード・ヒルズ・ロイヤルカナン・
  ユニチャーム・マース・ジェックス・ネスレ・ペティオ・デビフ … **主要大手はすべて0件**。
- 成分ページが取れたのは **6社・計12ページのみ**：
  ペットライン2 / ペットライブラリー3 / レッドハート4 / アダプトゲン1 / ビルバック1 / マルトモ1。

→ READMEの引き継ぎ所見（「requestsだけの静的クロールでは大手の栄養データは取れない」）を**実測で確定**。

### 2-2. 項目別 開示率（取れた12ページに対する割合）

| 項目 | 開示 | 率 |
|------|------|----|
| 原材料 | 12/12 | 100% |
| 水分 | 10/12 | 83% |
| 粗たんぱく質 / 粗脂肪 / 粗繊維 / 粗灰分 / カロリー | 9/12 | 75% |
| **リン** | **1/12** | **8%** |

唯一のリン開示はペットライン（リン 0.7%・タンパク質29.5%・340kcal）。実在値で妥当。

### 2-3. 撤退ライン判定（`02` の基準に照合）

> リン開示率 **8%** → `< 20%` ＝ **腎臓は見送り。体重管理（カロリー密度）を看板に切替**。

ただし **母数が小さく（12ページ）大手不在のtail偏重**なので、この判定は**暫定**。
カロリー密度（75%）・原材料（100%）は看板を張れる水準にある＝**体重管理ローンチは現実的**という
ポジティブな含意も同時に得られた。

## 3. JS対応PoC（④）— Playwrightでも「汎用巡回」では届かない

大手0件の主因はJS描画。ヘッドレスChromium（Playwright・トークンゼロ）でアイシアを再巡回したが
**トップからのBFSでは0件**のまま。診断の結果：

- アイシアの製品詳細は `/product/result?...` という**サイト内検索アプリ**の奥にあり、
  一覧→詳細がAJAXで読まれる。トップからリンクを辿る汎用巡回では詳細に到達しない。
- つまり大手は **Playwrightを使っても「サイト個別の導線対応（製品詳細URL列挙 or 内部API）」が必要**。
  画像化されている社があればさらにOCR（ローカル＝トークンゼロ）が要る。
