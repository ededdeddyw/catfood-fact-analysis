# (a) 第3段「腎臓」相談シート — 具体設計

> 悩み機能のうち**最もリスクが高く、最も価値が高い**のが腎臓。
> ここを「非診断のまま」成立させる設計。出力は「おすすめフード」ではなく **獣医師に持っていく比較ファクトシート**。
> 前提・全体思想は `README.md` を参照。

---

## 0. これは何をする画面？（非エンジニア向け）

飼い主が「うちの子は腎臓が心配」を選ぶと、腎臓に関係する栄養数値（リン・タンパク質など）を、
**メーカー公式の表示から転記・換算して1枚の比較表**にまとめる。
サービスは「これが良い」とは言わない。表を印刷して**かかりつけの獣医師に見せてもらう**ための準備ツール。

---

## 1. 設計の絶対原則（これを破ったら医療判断になる）

| # | 原則 | 理由 |
|---|------|------|
| 1 | **並べ替えない・点数化しない** | 「リンが低い順」も実質"低リン推奨"という判断＝NG。並びは**商品名順か価格順**のみ |
| 2 | **「良い／悪い」を一切付けない** | 数値と開示状態だけを出す。色分けで暗に良し悪しを示すのも避ける |
| 3 | **全項目に「不明・要確認」を許す** | 空欄を「含まれない」と断定しない（病院の4状態と同一） |
| 4 | **全数値に出典URL＋取得日** | フードは価格・成分が変わる。**日付のない数値は載せない** |
| 5 | **療法食は強制で注意書き** | 自己判断切り替えは健康を損なう。獣医師の指示前提を明示 |

## 3. 比較表の列設計（行＝商品、横並びで比較）

| 列 | 中身 | 表示ルール |
|----|------|-----------|
| 商品名／メーカー | そのまま | — |
| 種別 | ドライ／ウェット | **乾物量換算の前提**として必須 |
| **リン** | mg/100kcal（**乾物量・カロリー基準に換算**） | 開示あり→数値＋出典／なし→「公式に記載なし（要確認）」 |
| タンパク質 | %（乾物量換算） | 同上。**「低い＝腎臓に良い」とは書かない** |
| ナトリウム | %／mg | 同上 |
| 水分 | %（保証分析値・法定表示） | ほぼ全件取得可 |
| カロリー密度 | kcal/100g | 計算可。中立注記のみ（下記） |
| 主原料 | 原材料表の先頭数品目 | 公式表示の転記 |
| 療法食か | 「療法食」／「総合栄養食・一般食」 | 療法食なら固定注記を強制表示 |
| 出典・取得日 | URL＋YYYY-MM-DD | **必須**。古いものは「要再確認」 |

### 列ごとの注意

- **リン**：腎臓の食事管理で獣医が最も見る値。ただし保証分析値に**含まれないことが多い**（→ `02` で開示率を実測する）。
- **タンパク質**：「低タンパク＝腎臓に良い」は現代獣医学で議論があるため**サービスは立場を取らない**。数値だけ。
- **カロリー密度**：腎臓ケアでは「少食でも体重を維持できるか」が論点になるため数値は有用。ただし注記は「目安」に留める。
- **乾物量換算**：ウェットとドライを公平に比較するため全数値を乾物量基準に揃える。「水分を除いて比較しています」と明示。

## 4. 固定文言テンプレート（＝このプロダクトの本体）

病院の `wp_publish_label()` と同じ思想で、安全文言を辞書化する。

### 必ず表示する注記

- **療法食の行**：
  「これは療法食です。**獣医師の指示なく与えないでください。** 自己判断での切り替えは健康を損なう恐れがあります。」
- **リンの注記**：
  「腎臓の食事療法に関する数値の解釈は症例ごとに異なります。数値の良し悪しは判断せず、この表を獣医師にお見せください。」
- **空欄（不明）の注記**：
  「『記載なし』は『含まれない』という意味ではありません。メーカーが公開していない、という意味です。」
- **末尾CTA**：
  「📄 この比較メモを印刷／PDF保存」＋「最終的な判断は獣医師へ」

### 禁止表現／許可表現

| 禁止 | 許可・推奨 |
|------|-----------|
| 「腎臓の猫にはこのフード」 | 「公式サイトに『腎臓ケア』の記載があります（出典）」 |
| 「低リンだから安心」 | 「リン含有量は ◯◯mg/100kcal（保証分析値から乾物量換算）」 |
| 「○○は危険な原材料」 | 「リン含有量は公式に開示されていません（要確認）」 |
| 「これを食べれば数値が改善」 | 「療法食です。獣医師の指示のもとで与えてください」 |

## 5. データが無いときの挙動（4状態の徹底）

- リン開示が少数しかない場合でも、**空欄をでっち上げない**。
- 「リンを公式に開示している商品だけ」を絞り込む補助フィルタを用意（開示そのものを信頼性指標にする）。
- 開示率が低すぎる場合は腎臓シートを出さない判断もあり得る（撤退ラインは `02` で定義）。

---

## 6. この設計がMVVを満たす理由

- 出典・4状態・乾物量換算 → **信頼性**が口コミ／ランキングサイトより明確に上。
- 「並べ替えない・点数化しない」→ **医療判断を構造的に出せない**作り。
- 出力＝相談シート＝**「次の行動」**（構想の核）を満たす。
- アフィリ報酬と無関係な「開示の有無」で序列が決まる → ③アフィリ遮断とも整合。

---

## 7. 他の悩みへの横展開メモ

同じ「列分解＋相談シート」型は他の悩みにも使える。

- **体重管理（第2段）**：主役列＝カロリー密度・脂肪%・1日給与量→1日コスト。「太った猫向け」と言わず「低カロリー密度フード」。
- **ライフステージ（第1段）**：主役列＝法定の対象ステージ表示・カロリー密度。完全客観で最安全＝ローンチの出発点。
