## ① ドッグフードへの横展開（同一エンジン）

キャットフードと**まったく同じ観点**（成分・保証分析値・リン・出典・4状態・乾物量換算）で
ドッグフードもやりたい。猫が3ジャンル目の証明なら、犬は4ジャンル目で
「ペット領域の検証可能ファクトDB基盤」の再現性をさらに裏付ける。

**いま効く実務上の含意（重要）**:
- **ドッグフードのデータを捨てない。** 既に harvest しているメーカー（いなば・アイシア等）の
  sitemap は `/product/detail` で**犬猫混在**しており、`product_facts_raw.csv` には
  ドッグ製品も入り得る。猫で絞り込む前に**生データは犬も保持**しておく。
- `harvest_sitemap_products.py` のパターンを犬用に切り替えれば（例: 日本PF の dogfood sitemap、
  RC の `/jp/dogs/products/`）そのまま犬版オーディットが回せる。
- `catfood_nutrition_patterns.py` の抽出ロジックは犬猫共通で流用可能。

## ② オーダーメイド/オリジナルフードの会社への接続

既存製品で満足できない飼い主向けに、**オーダーメイド・オリジナル調合フードを作る会社**へ
つなげたい。そこの情報も収集・掲載対象にしたい。

**位置づけ**: ファクトDB（比較）の先にある「次の行動」の出口。
- 比較しても合う既製品が無い → **個別調合という選択肢**を提示できる＝相談シート思想と地続き。
- アフィリのランキングと違い「評価せず選択肢を広げる」ので製品人格(`docs/03`)と矛盾しない。
