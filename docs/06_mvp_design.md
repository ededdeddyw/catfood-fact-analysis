# MVP 設計 — 体重管理 ＋ 腎臓相談シート（実データで具体化）

> `docs/01-03` の思想（評価せず・順位を付けず・母集団宣言・出典必須・4状態・乾物量換算・非診断）を、
> `04` で取得した実データ（557→精緻化後の product_facts）に乗せて**最初に出す画面**を確定する。
> 結論: **リン開示率40%でGO**。第1段に体重管理（最安全）、同時に腎臓相談シート（GO）を両輪で出す。

---

## 0. 何を最初に出すか

| 機能 | 根拠 | 状態 |
|------|------|------|
| **体重管理ビュー**（カロリー密度） | カロリー/原材料/水分がほぼ全件開示。最安全・客観 | 第1ローンチ |
| **腎臓相談シート** | リン開示40%でGO。開示メーカーに絞れば成立 | 同時 or 直後 |
| ライフステージ | 法定表示ベース（別データ源）。後続 | 後続 |

どちらも **`docs/01` の「列分解＋相談シート」型**。スコア化・順位付けはしない。

## 1. 全機能で共通の作り（製品人格の実装）

- **母集団を宣言**：「ペットフード公正取引協議会 正会員79社のうち、公式に成分を開示している商品」。
  カバー率と未掲載を `04` のとおり正直に表示（②）。
- **猫のみ**（`species=cat`）。犬・「both」「dog」は除外（犬は将来 `docs/05`）。
- **4状態**：値あり=開示／無し=「公式に記載なし（要確認）」。空欄を「含まれない」と書かない。
- **出典必須**：各商品に公式URL＋取得日。
- **ランキングなし**：並びは商品名／価格／カロリー密度／開示項目数 の**ユーザー選択ソートのみ**。手数料は一切参照しない（③）。
- **乾物量換算（旗印）**：ウェット/ドライを公平比較するため水分を除いて換算（下記§4）。
- **非診断の固定注記**（`docs/01 §4` の文言をそのまま使用）。

## 2. 体重管理ビュー（第1ローンチ）

「太った猫向け」とは言わず **「カロリー密度の低い順に並べ替えられる一覧」** として提供。

| 列 | 中身 | 備考 |
|----|------|------|
| 商品名／メーカー | 公式表記 | — |
| 種別 | ドライ/ウェット | 水分から推定 or 表記 |
| **カロリー密度** | kcal/100g | `calorie_basis=per_100g` のみ密度として表示。`per_piece`（個包装おやつ）は「密度比較不可」と明示 |
| 脂肪%（乾物量） | 粗脂肪をDM換算 | — |
| 原材料 | 先頭数品目 | 公式転記 |
| 出典・取得日 | URL＋日付 | 必須 |

## 3. 腎臓相談シート（GO・`docs/01` の実装）

**リンを公式開示している商品だけ**に絞り込むフィルタを既定ON（開示そのものを信頼性指標にする）。

| 列 | 中身 | 表示ルール |
|----|------|-----------|
| 商品名／メーカー | 公式 | — |
| 種別 | ドライ/ウェット | 乾物量換算の前提 |
| **リン** | %（乾物量換算）。理想は mg/100kcal | 開示のみ。無→「公式に記載なし（要確認）」。良し悪しは書かない |
| たんぱく質 | %（乾物量） | 「低い=良い」とは書かない |
| 水分 | %（保証分析値） | ほぼ全件 |
| カロリー密度 | kcal/100g | per_piece は注記 |
| 療法食か | フラグ | 療法食なら固定注記を強制表示 |
| 出典・取得日 | URL＋日付 | 必須 |

固定注記・禁止/許可表現は `docs/01 §4` を踏襲（「この表を獣医師にお見せください」等）。

---

## 4. 乾物量換算（dry matter basis）

水分を除いた基準に揃える：

```
DM値(%) = as-fed値(%) ÷ (100 − 水分%) × 100
```

- 例: ウェット(水分76%)のたんぱく質9% → DM 37.5%。ドライ(水分10%)の30% → DM 33.3%。
  生表示では犬猫サイトが軒並み誤るところ、水分を除くと逆転する＝差別化の旗印。
- 水分が無い商品はDM換算しない（as-fed値に「※水分非開示のため換算不可」注記）。
- リンも同式でDM換算。`calorie_basis=per_piece` は密度算出から除外。

## 5. データフロー

```
data/product_facts_raw.csv        （精緻化抽出: species/calorie_basis/全角正規化/ブロック限定）
  └─ scripts/build_consult_sheet.py
       ├─ 猫のみ・成分ページのみ抽出
       ├─ as-fed → 乾物量換算（DM列を計算）
       ├─ per_piece を密度比較から除外フラグ
       └─ 出力:
            data/consult_sheet_cat.csv      （相談シートの素データ）
            prototype/consult/index.html    （静的プレビュー: ランキング無し・4状態・出典・DM・固定注記）
            prototype/consult/data.json
```

LLM不使用。静的HTMLなのでそのまま獣医に見せる印刷/PDFにも合う（`docs/01` の出力頂点）。
