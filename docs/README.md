# キャットフード ファクトベース分析サービス — 構想ドキュメント

> ⚠️ **これは pet-er.jp（病院ファクトDB）とは別の事業アイデアです。**
> 病院プロジェクトのコード・マスタ・MVV運用には混ぜません。本フォルダ（`docs/catfood_concept/`）内で完結する企画メモです。
> 病院プロジェクトで確立した「思想（4状態ラベル・出典必須・非診断・段階主義）」だけを横展開します。

---

## 一言コンセプト

キャットフードを、広告やランキングではなく **成分・原材料・栄養基準・価格・リスク情報のファクト**から分析する。
口コミ・推測スコアは出さない。判断材料と「次の行動」まで整理する。

## このサービスの戦略的な核

キャットフード単体の収益が目的ではない。

> **病院に続く2ドメイン目として「ペット領域の検証可能ファクトDB基盤」が再現することを証明する。**

1つ目=病院、2つ目=フード。同じエンジン（4状態ラベル・出典・段階リリース）が別ジャンルでも機能すれば、プラットフォーム仮説が立つ。ここが本質。

## 病院プロジェクトから引き継ぐ原則（不変）

| 原則 | フードでの意味 |
|------|----------------|
| **4状態ラベル** | 肯定（公式記載）／否定（公式記載）／不明・要確認／矛盾。「情報なし」を「含まれない」と混同しない |
| **一次ソース優先** | メーカー公式サイト＞パッケージ画像＞法定表示＞ECページ。混ぜない |
| **出典必須** | 全数値に出典URL＋取得日。フードは鮮度が命なので**日付なしは載せない** |
| **非診断** | 「この病気にはこのフード」を言わない。療法食は獣医師の指示が前提 |
| **段階主義** | 50商品→拡大。悩み機能もリスクの低い順（ライフステージ→体重管理→腎臓） |
| **過大評価しない** | 件数や網羅を盛らない。取得率は正直に出す（→ `02_data_feasibility_audit.md`） |

## MVV判定（この事業を採用する根拠）

| 唯一の問い | 判定 |
|------------|------|
| 信頼性が上がるか | ◎ 出典付き成分・法定表示・客観計算。アフィリのランキングへの明確なアンチテーゼ |
| 網羅性を客観証明できるか | △ 病院の「行政名簿JOIN」に相当する公的フード名簿がない。**母集団の宣言**で補う（要設計・②） |

→ 信頼性は満点級。**網羅性の客観証明だけが病院より弱い**ので、MVPで対象母集団を明示限定する。

## 差別化（敵＝アフィリのランキング記事）

キャットフード検索の上位はほぼ全部アフィリエイトのランキング（=最も嫌う口コミ・推測スコア）。
→ **ファクト＋出典＋ランキングなし**は SEO的に空白地帯。差別化が"実在"する。

最大の技術的旗印：**乾物量換算（dry matter basis）**。
ウェット（水分80%）とドライ（水分10%）を保証分析値の生数字で比べると全サイトが間違える。水分を除いて比較するだけで既存サイトに優位。

## 「悩み別」機能の設計思想（最重要）

「適合度（おすすめ度）」という言葉を捨てる。スコア化＝医療判断＝出してはいけない。

> **「判断する」のではなく「判断できる状態に準備する」プロダクト。**

各悩みを「計算できる客観指標」＋「4状態ファクト（開示の有無）」に分解して**そのまま見せる**。
出力の頂点は **「獣医さん相談シート」**（→ `01_kidney_consult_sheet.md`）。これで非診断と「次の行動」を両立する。

### 悩み機能の段階リリース（リスクの低い順）

1. **第1段：ライフステージ** — 法定表示・完全客観・最安全。ここからローンチ
2. **第2段：体重管理** — カロリー密度で勝負。「低カロリー密度フード一覧」とし「太った猫向け」とは言わない
3. **第3段：腎臓・特定健康懸念** — 開示ベース＋相談シート＋強い注意書き。一番難しく一番価値が高い（→ `01`）

## 詰め終わった論点（①②③）

- **①悩み別適合度** → スコア化しない。客観指標＋4状態に分解、出力は獣医相談シート（`01`）
- **②網羅性の定義** → 全網羅でなく「宣言した母集団へのカバー率＋未掲載レポート」（`03`）
- **③アフィリ遮断** → ランキングを作らない・表示順は手数料無関係・コードで分離（`03`）
