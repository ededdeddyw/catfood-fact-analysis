# 透明な条件マッチ — 「レコメンドしない」の進化（2026-06-23 方針転換）

> 当初の `01-03` は「評価せず・順位を付けず・おすすめを出さない」だった。
> しかし**ファクトだけでは飼い主は行動できず、サイトの価値が立たない**という気づきから、
> レコメンドを入れる。ただし**アフィリのランキングには戻さない**。捨てるのは「冷たさ」、
> 残すのは「透明性・非診断・手数料非連動」。

---

## 何を変え、何を変えないか

| 変える | 変えない（武器） |
|--------|-----------------|
| 「ファクトだけ淡々」をやめ、**目的に合う商品を提示**する | 評価の中身を隠すブラックボックス・スコアは作らない |
| 「こうなりたい人にはこれ」という**導線**を持つ | 総合○位／おすすめ度ランキングは作らない |
| 観点（目的）を増やす（体重・腎臓だけでなく） | 表示順に**手数料を一切使わない** |
| | 健康系の観点は**非診断＋獣医相談**を維持 |

## 中核の型：透明な条件マッチ

> **こうなりたい（目的）→ 見る客観指標を明示 → 条件を明示 → 合致商品＋"なぜ合うか（実値）"を全部見せる**

- ブラックボックスの点数ではなく、「この目的では**この指標**を見る／**この条件**で絞る」と明言する。
- 各商品に「なぜ合うか」＝**実際の数値**を併記（隠さない）。
- ＝ アフィリの「総合○位」とは別物（基準が見える・手数料無関係）。

## 観点とリスク段階

- **生活系（しっかり推奨可）**：体重管理／高たんぱく／水分を摂らせたい／穀物を避けたい／毛玉・繊維。
  → 「条件に合うのはこれ」と言い切ってよい（医療判断ではない）。
- **健康系（指標提示＋獣医併記）**：腎臓（リン）／尿路（マグネシウム・灰分）。
  → 「獣医が見る指標がこの範囲の商品」までにとどめ、「この病気にはこれ」とは言わない。
    療法食は獣医の指示が前提。相談シートに繋ぐ。

## 既存データで作れる観点（新規取得ゼロ）

| 観点 | 指標 | 条件 | 段階 |
|------|------|------|------|
| 体重を管理したい | カロリー密度 kcal/100g | 低い順 | 生活 |
| 高たんぱくにしたい | たんぱく質(乾物量) | 高い順 | 生活 |
| 水分を摂らせたい | 水分% / 種別 | ウェット | 生活 |
| 穀物を避けたい | 原材料 | 主要部に穀物表記なし(参考) | 生活 |
| 毛玉・繊維を増やしたい | 粗繊維% | 高い順 | 生活 |
| 尿路が気になる | マグネシウム% / 灰分% | 低い順・開示のみ | 健康(獣医併記) |
| 腎臓が気になる | リン(乾物量) | 低い順・開示のみ | 健康(獣医併記) |

## 実装

`build_site.py` に観点（GOALS）を定義し、`find.html`（目的から選ぶ）を生成。
目的を選ぶと「見る指標・条件・理由」を提示し、合致商品を実値付きで並べる。
健康系は獣医相談の注記を強制表示。`01-03` の禁止（順位・手数料・断定）は全て維持。
